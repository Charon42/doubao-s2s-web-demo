# 豆包端到端实时语音 S2S 网页 Demo 使用说明

这个 Demo 的链路是：浏览器采集麦克风音频并转成 16kHz mono PCM16，小端序 int16；前端通过 `/ws/call` 发给 FastAPI；后端连接豆包端到端实时语音大模型 WebSocket，并把前端 PCM 音频封装成火山自定义二进制协议发送给豆包。

## 需要准备

1. Python 3.10 或更高版本。
2. 火山引擎豆包端到端实时语音大模型权限。
3. 控制台中的 `App ID`、`Access Key`、`App Key`。
4. 确认模型版本和音色是否可用，例如：
   - `DOUBAO_MODEL_VERSION=1.2.1.1`
   - `DOUBAO_SPEAKER=zh_female_vv_jupiter_bigtts`

不要把真实密钥写进 `backend/.env.example`，只写进本地 `backend/.env`。

## 安装依赖

```powershell
cd "f:\job\ai翻译官\doubao-s2s-web-demo"
py -m pip install -r backend\requirements.txt
```

## 配置 .env

复制示例配置：

```powershell
Copy-Item backend\.env.example backend\.env
```

编辑：

```powershell
notepad backend\.env
```

需要填写：

```env
DOUBAO_WS_URL=wss://openspeech.bytedance.com/api/v3/realtime/dialogue
DOUBAO_APP_ID=你的AppID
DOUBAO_ACCESS_KEY=你的AccessKey
DOUBAO_RESOURCE_ID=volc.speech.dialog
DOUBAO_APP_KEY=你的AppKey
DOUBAO_MODEL_VERSION=1.2.1.1
DOUBAO_SPEAKER=zh_female_vv_jupiter_bigtts
DOUBAO_INPUT_SAMPLE_RATE=16000
DOUBAO_OUTPUT_SAMPLE_RATE=16000
```

旧的 `Authorization: Bearer`、`session.update`、`input_audio_buffer.append`、`response.create` 配置已经不再使用。

## 启动

```powershell
py -m uvicorn backend.main:app --host 127.0.0.1 --port 8000
```

浏览器打开：

```text
http://127.0.0.1:8000/
```

点击“开始通话”，允许浏览器使用麦克风。

## 调试

后端终端会打印：

- 浏览器 WebSocket 是否已连接
- 是否准备连接豆包 WebSocket
- 豆包 WebSocket 是否连接成功
- `.env` 是否读取到
- WebSocket URL 是否为空
- Access Key 是否读取到，脱敏显示
- 实际使用的 Header 字段名

如果页面连接后马上关闭，优先检查：

1. `backend/.env` 是否存在。
2. `DOUBAO_APP_ID`、`DOUBAO_ACCESS_KEY`、`DOUBAO_APP_KEY` 是否填对。
3. `DOUBAO_RESOURCE_ID` 是否为 `volc.speech.dialog`。
4. 当前账号是否开通该模型版本和音色。
5. 终端 traceback 和页面事件日志里的完整错误信息。
